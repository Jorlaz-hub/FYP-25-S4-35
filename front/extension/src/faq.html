FAQ & Help

Common questions and vulnerability explanations.

General Questions

Is my data secure?

Yes. This extension runs entirely on your device ("client-side").

  • No browsing data is sent to external servers.
  • Scan results are stored locally in your browser's storage.
  • You can clear all stored data at any time via the Settings menu.
What does the "Health Score" mean?

The score is a calculation (0-100%) based on three areas:

  • Structure: Usage of inline scripts and event handlers.
  • Security: Presence of security headers like CSP and HSTS.
  • Exposure: Data leaks, insecure forms, and third-party script risks.

Vulnerability Definitions

What is "Reverse Tabnabbing"?

This occurs when a link uses target="_blank" without rel="noopener". The newly opened page can access your original page via window.opener, potentially redirecting it to a malicious site.

Why are "Inline Scripts" bad?

Scripts written directly in HTML (<script>...</script>) are difficult to secure with a Content Security Policy (CSP). They make it harder to prevent Cross-Site Scripting (XSS) attacks because the browser cannot easily distinguish between your valid script and a malicious one injected by an attacker.

What is a CSRF Token?

A Cross-Site Request Forgery (CSRF) token is a secret value included in forms. It proves that the form submission originated from your actual website, not from a malicious link clicked by the user elsewhere.

What is "MIME Sniffing" (X-Content-Type-Options)?

Browsers sometimes try to "guess" the file type of a resource, ignoring what the server says. Attackers can use this to disguise malicious scripts as images or text files. The X-Content-Type-Options: nosniff header prevents this behavior.

Why are "Third-Party Scripts" a risk?

Third-party scripts are code loaded from other websites (like analytics or ad trackers). If the external site is hacked, attackers can inject malicious code into your site automatically. Our extension flags these so you can review if they are necessary and trusted.

What is "Subresource Integrity" (SRI)?

SRI is a security feature that prevents external scripts from being tampered with. It works by checking a cryptographic "fingerprint" (hash) of the file. If the file content changes (e.g., due to a hack), the browser blocks it. This extension warns you when external scripts are missing this protection.

What are "Hardcoded Secrets"?

This refers to sensitive information like API keys, passwords, or authentication tokens that are written directly into the code. This is dangerous because anyone viewing the page source can steal them. The extension scans for patterns that look like these leaked secrets.

What is "Template Injection"?

Server-Side Template Injection (SSTI) occurs when user input is unsafely embedded into a server template. We scan for markers like {{...}} or <%...%> which might indicate that internal server logic is being exposed to the client, potentially allowing attackers to take control.

Why is "Mixed Content" (HTTP vs HTTPS) flagged?

HTTPS encrypts the data between your browser and the site. If a page loads resources (like scripts or styles) over an insecure HTTP connection, attackers can intercept or modify that content. The extension penalizes pages that do not use HTTPS exclusively.

What is "Clickjacking" protection?

Clickjacking involves tricking a user into clicking something different from what they perceive, often by overlaying invisible frames. We check for headers like X-Frame-Options or CSP frame-ancestors which prevent your site from being secretly embedded inside other malicious sites.
